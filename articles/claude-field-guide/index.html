There is a particular frustration in having a powerful tool you don't fully understand. You use one corner of it, sense there is more, but can't quite see the shape of the whole. That is where most people are with Claude right now. They have the hammer. They don't know about the chisel, the saw, or the workbench.

This article is the map. Not a manual — a map. After reading it, you will know what each tool is, what it is good for, and how they connect.

I

The Seven Pieces

A plain-language introduction before the details

Before going deep on any single tool, here is the whole system in one paragraph. Claude Chat is the conversation. Claude Projects is the memory around the conversation. Claude Code is the agent in your terminal, doing real work on real files. Cowork is the same agent, but for non-coders, living inside the desktop app. Dispatch is the mobile remote control for Cowork, just released. Skills are specialised instruction packs you load into Claude so it can handle specific file types or workflows. CLAUDE.md is the config file that makes any of the above behave consistently inside your project.

Now each, in turn.

CH
Claude Chat
The conversation layer. claude.ai or the mobile app.
All plans

This is where most people start and, for many, where they stay. Claude Chat is a single conversation window — you type, Claude responds, you refine. It is the most approachable entry point and still deeply capable for writing, analysis, research, ideation, and editing.

What it cannot do: remember yesterday's conversation (unless you're in a Project), act autonomously on files, or complete a task while you step away.

WritingArticles, newsletters, scripts, emails, Substack drafts
ResearchSummarise PDFs, synthesise sources, explain complex topics
ThinkingBrainstorm, critique your own ideas, work through decisions
CreationGenerate artifacts, HTML pages, reports, image prompts
PR
Claude Projects
Persistent context across conversations.
Pro / Max

A Project is a folder that holds documents, custom instructions, and conversation history — all of which persist between sessions. When you open a Project, Claude already knows your style guide, your brand voice, the documents you've uploaded, and what you discussed last week.

Think of it as a permanent briefing that runs before every conversation. This is the difference between a contractor who shows up cold each morning and a collaborator who has been with you for months.

PublishingUpload your master brief once. Every session already knows your voice, format rules, and house style.
ResearchUpload your source PDFs. Ask questions across all of them without re-uploading.
ClientsOne project per client. Upload their brief, tone guide, and past deliverables.
CC
Claude Code
An autonomous agent that lives in your terminal.
Pro / Max

Claude Code is not an autocomplete tool. It reads your entire codebase, forms a plan, executes bash commands, edits multiple files, runs tests, catches failures, and corrects itself — all from a single natural-language instruction. As we wrote in the previous article in this series, it is closer to an operating system than a coding assistant.

You install it once in your terminal (npm install -g @anthropic-ai/claude-code), navigate to any project folder, and type claude. From that moment, you have an agent that can build, debug, refactor, and ship.

Recent additions make it significantly more powerful: Checkpoints auto-save your state before every change, so you can rewind with /rewind. Subagents let one agent delegate to another in parallel. Remote Control (launched February 2026) lets you monitor and direct a running session from your phone.

$ claude
> Build me a static site index.html that lists all HTML files in the /articles folder
# Claude reads the folder, writes the file, opens it for review
Site buildingBuild and push your GitHub Pages site from one instruction
AutomationWrite scripts that batch-process files, rename, organise, convert
PDF generationGenerate a formatted PDF from markdown notes automatically
CW
Cowork
Claude Code's power, without the terminal. For everyone.
Pro / Max

Cowork is what happens when Anthropic noticed that non-developers were using Claude Code to organise files, compile research, and draft documents — and decided to make that experience official. It lives inside the Claude desktop app (Mac and Windows), runs in an isolated virtual machine on your own computer, and gives Claude direct access to the folders you grant it.

The workflow is simple: open Cowork, point it at a folder, describe the outcome you want, review the plan, and step away. Claude works while you do something else. You come back to finished work.

You can also set scheduled tasks. "Every Monday morning, pull last week's Substack stats, format them into a one-page report, and save it to my Obsidian vault." That runs without you doing anything.

File organisationSort, rename, and structure months of accumulated downloads in minutes
Report generationTurn a folder of receipts or screenshots into a formatted spreadsheet
Content productionFeed it raw notes, get a first draft back in the correct format
Scheduled workWeekly briefings, monthly reports — set once, run forever
DI
Cowork Dispatch launched Mar 17
Your phone as a remote control for your desktop agent.
Pro / Max

Dispatch solves a genuine friction point: Cowork is powerful, but you had to be at your desk to use it. Dispatch breaks that constraint.

The setup takes sixty seconds. Open Cowork on your desktop, click Dispatch, scan the QR code with your phone. Done. From that moment, your phone is a walkie-talkie to your desktop. You send a prompt — "Organise my Q1 receipts and make a PDF summary" — your Mac receives it, executes the task locally in its sandboxed environment, and sends you back the result. Your files never leave your machine.

One important caveat: your computer must stay on and Claude Desktop must stay open. Dispatch is a remote control, not cloud computing. If your laptop sleeps, the session ends.

Morning briefWhile making chai: "Summarise yesterday's emails and my calendar for today"
On the moveKick off a long research task from your phone, come back to results
Content pipelineTrigger article drafts or report generation from anywhere
SK
Skills
Specialised instruction packs that extend Claude's native abilities.
All plans

A Skill is a folder containing a SKILL.md file — instructions, templates, and examples that teach Claude how to handle a specific task category to a professional standard. When Claude loads a Skill before working, it is not guessing at format. It knows exactly what a great Word document looks like, how to structure a PDF, what a well-formed PowerPoint requires.

Anthropic ships pre-built Skills for DOCX, XLSX, PDF, and PPTX. You can also build your own: a Skill for your newsletter template, a Skill for your video script format, a Skill for your PDF house style. Once built, every piece of content comes out consistent without re-explaining the format each time.

Document creationProfessional Word docs, PDFs, presentations that match your house style
Custom workflowsA publication-specific skill that encodes all your formatting rules
Cowork integrationSkills load automatically inside Cowork for relevant file types
MD
CLAUDE.md
The config file that makes any agent behave consistently.
Claude Code

CLAUDE.md is a plain Markdown file you place at the root of any project folder. Claude Code reads it at the start of every session. Whatever you write there — your tone preferences, your technical conventions, your output format requirements — becomes the ambient context for every action Claude takes.

It is not documentation for you. You already know these things. It is the briefing note you give a very capable new team member on their first day, except this team member reads it fresh every morning and never forgets it.

## Project: My Publication
# Read this before every session

Tone: warm, curious, accessible. Morgan Housel, not Wikipedia.
Never use em dashes.
All articles end with an image generation prompt.
Thumbnails: 14:10 ratio, painterly and warm.
Exports: save to /exports folder.
Brand voiceEncode your tone, style rules, and no-go phrases once, applied always
Project conventionsFile naming, output folders, tool preferences, commit message style
Multi-projectEach project folder gets its own CLAUDE.md — agents context-switch automatically

II

How They Fit Together

The same system, seen as a whole

A useful mental model: Claude Chat is where you think out loud. Projects is the memory that makes that thinking persistent. Claude Code and Cowork are where thinking becomes action. Skills are the craft standards your agent applies when it acts. CLAUDE.md is the standing orders that govern how it acts. Dispatch is the thread that keeps you connected to it all, even from your phone.

You are not using seven separate tools. You are building one system: a persistent, context-aware, increasingly autonomous collaborator that knows your work, your voice, and your goals — and keeps working even when you step away.

The practical implication: the tools compound. A Claude Code agent operating inside a well-structured project folder, with a good CLAUDE.md, with the right Skills loaded, is not the same creature as a bare Claude Code session. The briefing sharpens every action. Each layer you add multiplies the quality of the output.
